## Findings

### 1. ES-module conversion for `src/shared/` — **Action Required**

- **Evidence**: 15 dual-export CJS modules (37 `typeof module` sites); the
  shared classic-`<script>` lexical scope caused three real-boot-only defect
  classes across M06 F1–F3 (top-level const collisions), contained today by
  two vm-replay nets (163 lines) that need hand-extension per new page.
  Compensating machinery: 495-line `renderer-globals.d.ts`, 32-entry eslint
  globals block, hybrid require-or-global resolution in 3 modules.
- **Feasibility (probe-verified, Electron 42)**: `<script type="module">` with
  real relative imports works over `file://` (chrome path:
  contextIsolation:true, sandbox:false) AND over the privileged `goldfinch://`
  scheme (sandbox:true, strict CSP) — probes replicated production conditions
  including the exact CSP string. Node 22/24 `require()` of ESM works
  unflagged under `node --test`, matching the suite's exact require pattern.
  `contentTypeFor` already serves `.js` as `text/javascript`.
- **Recommendation**: pilot-gated conversion. Pilot = 2–3 low-fanout modules +
  one page, MUST include an order-sensitive dependency (module scripts are
  always deferred; current tags are not — ordering must be proven live).
  Sweep gated on pilot green. Then retire d.ts, eslint globals, vm nets,
  hybrid resolution. ~4–5 legs, risk low-medium.

### 2. `automation-find.test.js` real timers — **Action Required**

- **Evidence (measured)**: full suite 5.21s wall with the file, **1.09s
  without** — one file is ~80% of suite wall-clock. Three tests are 91% of the
  file's cost (two real-timer retry-exhaustion tests: 2502ms + 1560ms; one
  explicit 520ms sleep at line 198). Five flights of flat ~5.0s internal
  duration are fully explained; no flakes/skips anywhere (0/0 across all M06
  checkpoints, fresh run confirms 1283/1283 at 5046ms).
- **Recommendation**: `node:test` MockTimers on the ~5-6 real-timer tests; a
  clock-injection seam in the engine's find-retry logic if needed. The F3
  debrief's 60–75% estimate is confirmed and slightly exceeded (~80%). 1 leg.

### 3. `main.js` growth — **Pass (accepted)**

- **Evidence**: 2545 → 2724 (+179, +7%) across M06. Composition verified:
  thin `registerJarIpc` glue (heavy logic already extracted), `INTERNAL_PAGES`
  data-table entries (documented as intentionally main-resident), a ~40-line
  dev-only auto-mint block whose decision logic already lives in
  `src/shared/automation-dev.js`, three broadcast-invariant fixes.
- **Recommendation**: accept; no extraction-ready seam. Keep the line-count
  watch as carried debt. 0 legs.

### 4. `{ok:false}` result-contract asymmetry — **Advisory**

- **Evidence**: `handleClearData` returns bare `{ok:false}` on every failure
  branch; `handleWipe` carries `error` on exactly one branch (line 213) — the
  asymmetry exists between AND within handlers. The renderer never reads
  `result.error` (static `failNote` only), so the field is currently dead data.
- **Recommendation**: standardize `{ok:false, error}` across both handlers'
  branches + unit tests, before history adds callers to the contract. Renderer
  display is optional follow-on. 1 small leg.

### 5. CLAUDE.md pattern promotions — **Advisory**

- **Evidence**: all three absent (grep-confirmed): uniform focus rule,
  `action:rowId` confirm-transition key, DD10(b) checklist widened to
  preload-bridge declares. Recurrence already happened once as predicted (F5
  leg 1's d.ts catch).
- **Recommendation**: land after the ESM decision — ESM retires 2 of 4
  DD10(b) items, so writing first wastes work. Placements identified (focus
  idioms section; new Patterns subsection; direct DD10(b) edit). 1 doc leg.
