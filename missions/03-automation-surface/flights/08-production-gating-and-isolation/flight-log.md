### Leg 5 — port-free-fallback (DD6) — **landed**
- 2026-06-17 — Scoped `GOLDFINCH_MCP_PORT` to **dev-only** and gave the setting/default path a **free-port fallback** that retries the real `listen` on EADDRINUSE, while capturing + surfacing the **actually-bound** port — never overwriting the persisted `automationPort`.
- **`resolvePort` stays pure (`mcp-server.js`):** added `resolvePort(getSettings, { honorEnv = true } = {})`; the `GOLDFINCH_MCP_PORT` read is now gated on `honorEnv`. Default `true` keeps all 9 existing `automation-port.test.js` cases + the zero-arg call green. No `app.isPackaged` / electron inside the function — the call site supplies the decision.
- **`createMcpServer` strict-vs-fallback detection:** accepts `opts.honorEnv` (default true). Reads the env **once** at construction: `explicit = Number.isInteger(opts.port) && opts.port > 0`; `envUsed = !explicit && valid GOLDFINCH_MCP_PORT (when honorEnv)`; `preferred = explicit ? opts.port : resolvePort(getSettings, { honorEnv })`; `strict = explicit || envUsed`. Holds `let boundPort = preferred` and the returned object now exposes `get port()` (replacing the fixed `port` value) — with a comment that callers must read `mcpServer.port` **live**, not destructure.
- **`start()` retry loop:** fresh `http.createServer(onRequest)` per attempt (a failed attempt never bound → discarded, no leak). On success: set `boundPort`, keep `started`/`httpServer`, resolve. On EADDRINUSE: **strict** → reset `started=false`/`httpServer=null` + reject loudly (no move); **fallback** → pick the next candidate via `freePortInRange(attempt + 1)` (advisory) then retry the **real** `listen` (the bind is the authority — defeats the probe's TOCTOU), capped at ≤20; on `null`/exhaustion reset + reject "no free port found". Non-EADDRINUSE → reset + reject. The fallback is **ephemeral** — `start()` never persists `automationPort`.
- **Build/mode-aware EADDRINUSE hint (`eaddrinuseMessage`):** dev env-strict mentions `GOLDFINCH_MCP_PORT` (must bind exactly); explicit-port strict gives a plain in-use message; fallback-exhausted points at the **Settings** port control (and on a packaged build — `honorEnv:false` — drops the "free a port" env framing since the env is ignored).
- **`main.js`:** `startMcpServerInstance` now passes `honorEnv: !app.isPackaged` into `createMcpServer`, keeps the pre-start `mcpStatus.port` (attempted, for the failure UI), and adds `mcpStatus.port = mcpServer.port` **after** `await mcpServer.start()` resolves (the bound port — may differ in fallback). `currentAutomationStatus`'s would-be branch now resolves `resolvePort(() => settings, { honorEnv: !app.isPackaged })`. No `settings.set('automationPort')` in the fallback path.
- **Tests (`automation-port.test.js`):** added `honorEnv:false` (env set but ignored → setting/default), `honorEnv:false` no-setting → default, `honorEnv:true` confirmation, and a zero-arg `resolvePort()` check. Added two `start()`-level integration tests: (a) **fallback** — occupy an OS-ephemeral port via `net.createServer().listen(0)` (NOT the fixed 7790/7791/7792 the mcp-server suite uses), start preferring it in fallback mode, assert it binds a **different** free port, `server.port` reflects it, `automationPort` **not** written; (b) **strict** — occupy an ephemeral port, start strict (explicit `opts.port`), assert `start()` rejects and does not move. Both wrap occupier AND started server in `try/finally` close; the ≤20 cap guarantees loop termination. **No hang.**
- **Gates:** `npm test` **732 pass / 0 fail** (was 726); `npm run typecheck` clean; `npm run lint` clean.
- **DEFERRED → leg 7 (verify-integration, packaged build):** live verification of (1) **two instances back-to-back both bind** (second falls back to a free port — the F7 mission Known Issue), (2) **dev env-strict hard-fail** (`GOLDFINCH_MCP_PORT=<taken>` → surface unbound, `mcpStatus.error` set, app keeps running), and (3) **packaged-build env ignored** (env set but port comes from `automationPort` + fallback).
