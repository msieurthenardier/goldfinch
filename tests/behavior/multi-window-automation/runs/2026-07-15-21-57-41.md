defect. The "siblings are tracked" argument is true but doesn't discriminate: they
are tracked because they belong to *landed* flights. And the fresh-clone argument
is mis-timed — on a fresh clone of `main` the **spec doesn't exist either**; spec
and fixture are the same commit's payload, so there is no window where a run sees
one without the other.

> ### ⚠️ THE ACTIONABLE RESIDUE — a real hazard the erratum was hiding
> **Untracked files are exactly what `git commit -a` misses.** If the flight-end
> commit uses `-a` or `git add -u`, it **silently drops** the fixture set, the
> spec, the run logs — **and four product source files**. Erratum 1 would
> re-materialize and take `window-census.js` with it. **The flight commit must
> explicitly `git add` the untracked paths and verify `git status --porcelain` is
> clean afterward.**
