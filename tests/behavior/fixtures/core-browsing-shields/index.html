CORE BROWSING + SHIELDS FIXTURE

core-shields-fixture-loaded

Purpose

This is a deliberate Shields test fixture for the core-browsing-shields behavior test. It exercises two Shields enforcement paths in the running Goldfinch app:

  1. Tracker block — the page requests google-analytics.com/analytics.js, a domain in Goldfinch's TRACKERS map (category: analytics). Shields (block strategy) must cancel that request via session.webRequest.onBeforeRequest.
  2. Tracking-param strip — load this page with ?utm_source=test&q=keep in the URL; Shields must strip utm_source and preserve q=keep via a redirect on the top-level navigation.

Tracker script (deliberate — Shields should block this)

The script tag below requests https://www.google-analytics.com/analytics.js. This domain is registered in src/main/trackers.js as category analytics. When Shields are on (default), Goldfinch cancels this request via session.webRequest.onBeforeRequest. The block is the observable this fixture is designed to produce — the Executor asserts ≥1 blocked tracker in the privacy panel after loading this page.

Tracking-param strip note

Load this page as http://127.0.0.1:8080/?utm_source=test&q=keep. Shields strips utm_source and preserves q=keep. Observe the address-bar / webview src URL after load — it should show ?q=keep only.
