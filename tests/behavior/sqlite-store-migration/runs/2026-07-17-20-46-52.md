- **Admin-tier apparatus evolution (the run's biggest operational finding)**:
  the session-registered goldfinch MCP tools ride a fixed jar-scoped key and
  cannot be re-registered mid-session; admin ops needed a different channel.
  The Orchestrator's first proposed mechanism (`curl -H "Bearer $KEY"`) was
  **refused by the Executor with a correct security analysis** (shell
  expansion lands the key in curl's argv → `/proc` visible), and its refusal
  was accepted. The final mechanism — used for the rest of the run — is a
  one-shot Node script over the project's own `scripts/lib/mcp-client.mjs`
  (`connectAutomation({key})` + `parseDevMintLine()`), key confined to a
  function argument in one short-lived process: never argv, env, disk, or
  stdout. **Recommend this as the documented pattern for future keyed runs.**
- **Premise correction for future specs**: MCP `openTab` cannot open
  `goldfinch://` internal pages under ANY identity (untrusted-branch
  `isSafeTabUrl` rejects the scheme by design; provenance is the call site —
  `renderer.js:1370-1374`). The sanctioned path is `getChromeTarget` +
  `evaluate` of the FD-approved chrome globals (`kebabActionSettings()`,
  `openJarsPage()`, `createTab(url, null, {trusted:true})` —
  `renderer.js:4160-4194`). Also confirmed: internal-session evaluate
  exclusion applies uniformly to all three internal pages.
- **Spec wording**: step 7's "leave `-wal`/`-shm`" should read "if present" —
  clean shutdown checkpoints them away.
